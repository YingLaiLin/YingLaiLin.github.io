泛型

什么是泛型

泛型即”参数化类型”.

泛型的使用

泛型类

类或接口: 关键字 + 类名 或 接口名 + <E>, E 为实际类型参数.

泛型方法

为什么要使用泛型?

  • 类型安全

    • 可以在编译时检查类型安全, 而非将其拖到运行期
    • 无需强制进行类型转换, 避免出错

  • 提高代码的灵活度

    • 同样的代码无需重复书写

泛型的缺点

Java 的泛型是伪泛型, 为了去兼容 Java 5 之前的旧代码(raw type), 因而其只在编译时去执行类型检查和类型推断, 而在编译为字节码之后并不存储类型信息, 也就是说, 其类型会被擦除.也因此会导致其存在一些问题.

  • 泛型类共用 class 对象.
  • 静态变量被泛型类的所有实例共享

  • 泛型类无法扩展 Throwable, 也无法在 catch 中使用.

  • 泛型的对象不能是基本数据类型

泛型的转换

泛型会进行类型擦除, 但是并不是都是转换为 Object 类型.

代码的坏味道

  2. 使用数组而非泛型列表
