泛型

什么是泛型

泛型即”参数化类型”.

泛型的使用

泛型类

类或接口: 类 或 接口 + <E>, E 为实际类型参数.

泛型方法

为什么要使用泛型?

  • 类型安全
    • 可以在编译时检查类型安全, 而非将其拖到运行期
    • 无需强制转换, 避免出错
  • 提高代码的灵活度
    • 同样的代码无需重复书写,

泛型的缺点

Java 的泛型是伪泛型, 为了去兼容 Java 5 之前的旧代码(raw type), 因而其只在编译时去执行类型检查和类型推断, 而在编译为字节码之后并不存储类型信息, 也就是说, 其类型会被擦除.也因此会导致其存在一些问题.

  • 泛型类共用 class 对象.
  • 静态变量被泛型类的所有实例共享
  • 泛型不可用于 Java 的 catch 语句

数组是协变的

这意味着如果 Sub 是 Super 的子类型, 则 Sub[] 是 Super[] 的子类型. 如果这样的话, 向其加入一个新的子类对象 Sub1, 在编译期间不会报错, 而在运行时会出现 ArrayStoreException 错误.

  1. 禁止创建泛型数组

SubTyping(子类型化)

List<String> 是 List 的一个子类型, 而不是 List<Object> 的子类型.

PECS

PECS的意思是 Producer Extends Consumer Super，简单理解为如果是生产者则使用Extends，如果是消费者则使用Super.

这里的生产者, 就意味着你要将数据提供给消费者消费, 也就是数据能够被取出.

这里的消费者, 就意味着你要将数据拿走消费, 也就是数据能够被存储在新的数据结构中.
