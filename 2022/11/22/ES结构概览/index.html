ES 结构概览

一个 ES Index 在集群模式下，由多个 Node （节点）组成。每个节点就是 ES 的Instance (实例)。每个节点上会有多个 shard （分片）.

与 Mongo 中 shard 的区分
这里要与 mongo 的 shard 区分, mongo 中 shard 就是节点的一种, 而在 es 中 shard 是按照集合维度定义的.

mongo: shard 上有多个集合, 每个集合又分布在不同的 shard 上.(节点维度)
ES: 某个 shard 上只有一个集合, 集合有多个分片, 对于特定的集合, 其 shard 分布在不同的 node 上.(集合维度)


如上图所示, P1 P2 是主分片, R1 R2 是副本分片每个分片上对应着就是一个 Lucene Index（底层索引文件）. 也就是说, 一个ElasticSearch的Shard 本质上是一个Lucene Index。

Lucene Index 是一个统称 由多个 Segment （段文件，就是倒排索引）组成。每个段文件存储着就是 Doc 文档。commit point 记录了所有 segments 的信息.

理解 segment

在Lucene里面有很多小的segment，我们可以把它们看成Lucene内部的mini-index.
