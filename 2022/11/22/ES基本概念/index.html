ES 基本概念

一个 ES Index 在集群模式下，由多个 Node （节点）组成。每个节点就是 ES 的Instance (实例)。每个节点上会有多个 shard （分片）.

与 Mongo 中 shard 的区分
这里要与 mongo 的 shard 区分, mongo 中 shard 就是节点的一种, 而在 es 中 shard 是按照集合维度定义的.

mongo: shard 上有多个集合, 每个集合又分布在不同的 shard 上.(节点维度)
ES: 某个 shard 上只有一个集合, 集合有多个分片, 对于特定的集合, 其 shard 分布在不同的 node 上.(集合维度)

ES 节点类型[1]

ES 集群中的节点类型包含:

  • master # 主节点
  • data # 通用数据节点
  • data_content # 数据目录节点
  • data_hot # 热点数据节点
  • data_warm # 中温数据节点
  • data_cold # 冷数据节点
  • data_frozen # 封存数据节点
  • ingest # 数据摄入节点
  • ml # 机器学习节点
  • remote_cluster_client # 远程候选节点
  • transform # 转换节点

注意点:

master - 主节点[2,4]

概念:主节点一般指的是集群中活跃的主节点，每个集群中只能有一个。

  • 主节点负责集群范围内的轻量级操作

    • 例如创建或删除索引、跟踪哪些节点是集群的一部分
    • 决定将哪些分片分配给哪些节点
    • 主节点必须有一个path.data 目录, 用于存储集群元数据

  • 候选主节点(master-eligible node) 可以参与主节点的选举, 其作用主要在于当主节点宕机或发生故障脱离集群时，参与选举成为新的主节点，从而避免集群无主。

  • 任何不是仅投票节点的主合格节点都可以通过主选举过程选举成为主节点

Dedicated master-eligible node 专用主节点

为了避免选出的主节点因为其他任务超载, 而将所有符合 master 条件的节点配置为专用的 master 节点, 这些节点只具有 master 角色.

符合主节点条件的节点仍然能充当协调节点, 当集群包含多个节点时,通常建议使用专用的主节点.

Voting-only master-eligible node 仅投票主节点

概念 : 仅投票主节点仍然会承担与其他主节点相同的工作, 只是其不会担任集群主节点. 和真正的主节点相比较，他们可能需要需要较少的堆和一个不太强大的 CPU. 仅投票节点的意义是在出现平票的时候投出关键票（决胜票）

由于候选主节点通常都是专用的, 由于其可能成为主节点, 因此候选主节点也不应承担 data 角色的功能, 这样会造成资源浪费. 而将其配置为仅投票节点, 让其承担 data 角色, 避免资源浪费.

    • 索引数据的创建、修改、删除、搜索、聚合

  • 热层必须能够快速进行读写操作，并且需要更多的硬件资源（例如 SSD 驱动器）

  • 冷数据节点的配置可以比中温层的配置更低

重要参数

index
来控制当前字段是否被索引

默认为 true，如果设为 false，则该字段不可被搜索.

index_options
用于控制倒排索引记录的内容.

  • doc：只记录 doc id
  • freqs：记录 doc id 和 term frequencies
  • positions：记录 doc id、term frequencies 和 term position
  • offsets：记录 doc id、term frequencies、term position 和 character offects

text 类型默认配置为 positions，其他类型默认为 doc，记录内容越多，占用存储空间越大。

字段类型

字符串类型

text
text 类型适用于需要被全文检索的字段，例如新闻正文、邮件内容等比较长的文字，text 类型会被 Lucene 分词器（Analyzer）处理为一个个词项，并使用 Lucene 倒排索引存储，text 字段不能被用于排序

keyword
适合简短、结构化字符串，例如主机名、姓名、商品名称等，可以用于过滤、排序、聚合检索，也可以用于精确查询。

数值类型

数字类型分为 long、integer、short、byte、double、float、half_float、scaled_float。

数字类型的字段在满足需求的前提下应当尽量选择范围较小的数据类型，字段长度越短，搜索效率越高.

对于浮点数，可以优先考虑使用 scaled_float 类型，该类型可以通过缩放因子来精确浮点数，例如 12.34 可以转换为 1234 来存储。

日期类型

在 ES 中日期可以为以下形式：

  • 格式化的日期字符串，例如 2020-03-17 00:00、2020/03/17
  • 时间戳（和 1970-01-01 00:00:00 UTC 的差值），单位毫秒或者秒
    即使是格式化的日期字符串，ES 底层依然采用的是时间戳的形式存储。

如上图所示, P1 P2 是主分片, R1 R2 是副本分片每个分片上对应着就是一个 Lucene Index（底层索引文件）. 也就是说, 一个ElasticSearch的Shard 本质上是一个Lucene Index。
