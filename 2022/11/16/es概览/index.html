ES 概览

基本概念

ElasticSearch是一款非常强大的、基于 Lucene 的开源搜索及分析引擎, 可以用于构建近实时的分布式搜索分析引擎.

其内部通过使用 RESTful API, 以隐藏 Lucene 的复杂性.

与关系型数据库的概念对比
| 关系型数据库 | ES |
| ———– | ———– |
| 数据库 | 索引(index) |
| 表 | 类型(type, 6.0.0 废弃) |
| 行 | 文档(document) |
| 列 | 字段(field) |
| 表结构 | 映射(mapping) |
| 索引 | 反向索引(不确定?) |

使用场景

  • 搜索类场景

    • 网站搜索
    • 垂直搜索
    • 代码搜索

  • 分析类场景

    • 日志管理与分析
    • 安全指标监控
    • Web抓取舆情分析

match_all表示查询所有的数据，sort即按照什么字段排序

查询参数解释:

  • _source 用于指示是否查询只需要的字段[3]

    • false 标识返回结果中无 _source 内容

    • 使用数组指定多个字段 “_source”: [“name”,”level”]

    • 使用 include/exclude 来包含显示或者排除字段

返回结果的字段解释:

_shards[1]
提供关于集合相关的分片信息

  • _shards.total 分片总数(包含主备)
  • _shards.sueeceeful 需主备分片节点数

通用字段[2]

  • took
    • Elasticsearch 运行查询所花费的时间（ms)
    • 具体为从接收请求到协调节点准备发送数据
    • 不包含
      • 发请求到 es 的时间
      • 在搜索线程池排队等待的时间
      • 发送数据到客户端的时间
  • timed_out
    • bool 字段, 标识搜索请求是否超时

hits
包含返回的文档集合和元信息(metadata)

  • total

    • value 匹配的文档数
    • relation
      • eq 指示是精确匹配
      • gte 指示为下界

  • max_score

    • 返回结果中文档最匹配的评分, 如果为空, 则说明未按照 _score 排序

  • hits

    • _index 包含该文档的集合
    • _score 标识返回文档的相关性(32 位浮点数, 使用match_all时不适用)
    • _source 集合中的文档(JSON 格式)
