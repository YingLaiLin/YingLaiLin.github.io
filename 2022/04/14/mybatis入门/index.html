MyBatis 入门

什么是 MyBatis

  • MyBatis 是一款优秀的持久层框架，它支持自定义 SQL、存储过程以及高级映射。
  • MyBatis 免除了几乎所有的 JDBC 代码以及手动设置参数和获取结果集的工作
  • MyBatis 可以通过简单的 XML 或注解来配置和映射原始类型、接口和 Java POJO（Plain Old Java Objects，即实体类）为数据库中的记录。

概念区分: 持久层是用于持久化工作的代码块,即 Dao 层, 同 Service 层, Controller 层. 持久化即将瞬时状态的数据持久化到数据库或者外部存储中, 避免数据因断电丢失.

MyBatis 的优势

  • sql 写在 xml 中, 便于统一管理与优化, 不会影响应用程序或者数据库的现有设计
  • 通过提供 DAO 层解决 sql 与代码间的耦合, 分离业务逻辑和数据访问, 可维护性强
  • 支持编写动态 sql
  • 提供对象关系映射标签, 支持对象与数据库的 ORM(Object Relational Mapping) 字段关系映射
  • 提供对象关系映射标签, 支持对象关系组建维护

核心对象

SqlSessionFactoryBuilder

  • SqlSessionFactoryBuilder 用于从 XML 文件中创建 SqlSessionFactory 实例
  • 一旦创建了 SqlSessionFactory，其就可以被抛弃, 因此 SqlSessionFactoryBuilder 实例的最佳作用域是方法作用域（也就是局部方法变量）
  • 可以重用 SqlSessionFactoryBuilder 来创建多个 SqlSessionFactory 实例，但最好还是不要一直保留着它，以保证所有的 XML 解析资源可以被释放给更重要的事情

SqlSessionFactory

  • SqlSessionFactory 一旦被创建就应该在应用的运行期间一直存在, 其不应当被丢弃. 因此其最好通过单例或者静态单例模式实现.
  • SqlSessionFactory 不应被多次重建, 其最佳作用域为应用.

SqlSession

  • 每个线程都应该有它自己的 SqlSession 实例, 并且 SqlSession 的实例不是线程安全的, 也因此不应被共享.
  • 其最佳的作用域是请求或方法作用域, 不可将其放在静态域或者任何托管作用域中.
